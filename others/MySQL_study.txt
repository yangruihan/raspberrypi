##类型：
CHAR(n)             长度为n的定长字符串
VARCHAR(n)          最大长度为n的变长字符串
INT                 长整数（也可以写作INTEGER）
SMALLINT            短整数
NUMERIC(p,d)        定点数，由p位数字（不包括符号、小数点）组成，小数点后面有d位数字
REAL                取决于机器精度的浮点数
Double Precision    取决于机器精度的双精度浮点数
FLOAT(n)            浮点数，精度至少为n位数字
DATE                日期，包含年、月、日，格式YYYY-MM-DD
TIME                时间，包含一日的时、分、秒，格式为HH:MM:SS

##SQL 使用经验：
1.注意删除、修改时的级联操作
2.编写完SQL后，要在DBMS中测试SQL语句
3.使用SQL进行查询时，为了提高查询效率，应当对SQL语句进行查询优化，尽量避免全表查询。
	(1)对使用非常频繁的数据建立索引
	(2)尽量避免在where子句中使用 != 或 <> 符号，或者使用 in 或者 not in 表达式，这些都会导致全表查询
	(3)采用分页查询，一次只查询一部分数据
4.事务执行时，所有元子操作应该同时成功或同事撤销

数据库复习重点：

1.理解概念模型：
	把现实世界中的客观对象抽象为某一种信息结构，这种信息结构并不依赖于具体的计算机系统，不是某一个DBMS支持的数据类型，而是概念级的模型。	

	表示方法：E-R 图（Entity Relationship Approach）

2.强制存取控制（MAC）方法：
	当某一用户（或某一主体）以标记label注册入系统时，系统要求他对任何客体的存取必须遵循如下规则：
	（1）仅当主体的许可证级别大于或等于客体的密级时，该主体才能读取相应的客体
	（2）仅当主体的许可证级别等于客体的密级时，该主体才能写相应的客体

3.触发器的类型：
	触发器按照所触发动作的间隔尺寸可以分为行级触发器（FOR EACH ROW）和语句级触发器（FOR EACH STATEMENT）
	例如：UPDATE TEACHER SET Deptno = 5;  // 其中表 TEACHER 有1000行
	如果是语句级触发器，那么触发动作只发生一次，如果是行级触发器，那么触发动作执行1000次；

4.模式分解：
	并不是规范化程度越高，模式就越好，而必须结合应用环境和现实世界的具体情况合理地选择数据库模式。
	求一个属性关于函数依赖集的闭包

5.事务的特性：
	（1）原子性
		事务是数据库的逻辑工作单位，事务中包括的诸操作要么都做，要么都不做
	（2）一致性
		事务执行的结果必须是使数据库从一个一致性状态变到另一个一致性状态
	（3）隔离性
		一个事务的执行不能被其他事务干扰
	（4）持续性
		持续性也称永久性（Permanenece），指一个事物一旦提交，它对数据库中数据的改变就应该是永久性的

6.并发控制概述
	（*背*）为了保证事务的隔离性和一致性，DBMS需要对并发操作进行正确调度，这些就是数据库管理系统中并发控制机制的责任。

7.数据库完整性
	数据的完整性是为了防止数据库中存在不符合语义的数据，也就是防止数据库中存在不正确的数据。
	（1）实体完整性
	（2）参照完整性
	（3）用户定义完整性

8.4NF
	（*背*）4NF就是限制关系模式的属性之间不允许有非平凡且非函数依赖的多值依赖

9.撤销未完成的事务
	恢复操作就是要撤销故障发生时未完成的事务，重做已完成的事务

10.层次模型的缺点：
	（1）现实世界中很多联系是非层次性的，如结点之间具有多对多联系
	（2）一个结点具有多个双亲等，层次模型表示这类联系的方式很笨拙，只能通过引入冗余数据（易产生不一致性）或创建非自然的数据结构（引入虚拟结点）来解决
	（3）查询子女结点必须通过双亲结点
	（4）由于结构严密，层次命令趋于程序化
